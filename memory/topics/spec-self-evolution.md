# 规范自检与进化机制

**禁止等用户指出问题才修复规范。**

## 触发条件

1. **执行失败** → 反推是否规范模糊
2. **用户指出问题** → 立即更新规范 + 记录教训
3. **新增自动化流程** → 同步更新规范
4. **每周 PCEC** → 审计规范完整度

## 检查清单

### 1. 可操作性检查

- ❌ 模糊词：启动、处理、推进、优化、改善
- ✅ 具体词：生成文件、调用 API、执行命令、更新字段

### 2. 触发条件检查

- ❌ 谁？何时？如何？不明确
- ✅ 明确的执行者、时间点、触发条件

### 3. 闭环验证检查

- ❌ 只有开始，没有结束
- ✅ 执行 → 验证 → 标记完成

### 4. 失败处理检查

- ❌ 没说阻塞怎么办
- ✅ 记录原因 → 备选方案 → 继续
