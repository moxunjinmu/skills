# 任务防丢失机制

## 核心原则

### 1. 外部化 > 内部化

永远不要相信自己的记忆，一切必须外部化。

- 答应任务 → 立即写入文件（`memory/YYYY-MM-DD.md`）
- 执行中状态 → 写入文件（`logs/task-status.json`）
- 完成结果 → 写入文件（`memory/YYYY-MM-DD.md`）

### 2. 承诺即执行

答应的事马上做，不做完不发「搞定」。

- 简单任务（<2 分钟）→ 当场完成
- 复杂任务 → 立即派子 agent，等返回再确认
- 阻塞任务 → 先记录阻塞原因，再报告

### 3. 状态机管理

任务不只是「完成/未完成」，有完整生命周期。

```
⏳ 进行中 → ✅ 完成 / ❌ 失败 / 🚫 阻塞 / ⏸️ 暂停
```

### 4. 会话启动检查

新会话必须检查未完成任务：

1. 昨天的 memory 文件（搜索「⏳」「❌」「🚫」「⏸️」）
2. `logs/task-pending.flag`（心跳生成的任务标记）
3. `logs/task-status.json`（任务状态详情）
